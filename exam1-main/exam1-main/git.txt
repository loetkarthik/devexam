DEVOPS WORKFLOW (GIT + DOCKER + GITHUB ACTIONS)

-----------------------------------
FILE STRUCTURE
-----------------------------------
project/
│── app.py
│── Dockerfile
│── requirements.txt
└── .github/
    └── workflows/
        └── ci.yml

-----------------------------------
STEP 1 – CREATE PROJECT
-----------------------------------
mkdir project
cd project

-----------------------------------
STEP 2 – CREATE PYTHON APP
-----------------------------------
cat <<EOT > app.py
print("Hello Docker CI")
EOT

-----------------------------------
STEP 3 – CREATE DOCKERFILE
-----------------------------------
cat <<EOT > Dockerfile
FROM python:3.10
WORKDIR /app
COPY . .
CMD ["python", "app.py"]
EOT

-----------------------------------
STEP 4 – CREATE REQUIREMENTS FILE
-----------------------------------
touch requirements.txt

-----------------------------------
STEP 5 – CREATE GITHUB ACTIONS WORKFLOW
-----------------------------------
mkdir -p .github/workflows

cat <<EOT > .github/workflows/ci.yml
name: Docker CI

on:
  push:
    branches: [ "main" ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v3

    - name: Login Docker Hub
      uses: docker/login-action@v2
      with:
        username: ${{ secrets.DOCKER_USERNAME }}
        password: ${{ secrets.DOCKER_PASSWORD }}

    - name: Build Image
      run: docker build -t ${{ secrets.DOCKER_USERNAME }}/myapp .

    - name: Push Image
      run: docker push ${{ secrets.DOCKER_USERNAME }}/myapp
EOT

-----------------------------------
STEP 6 – INITIALIZE GIT
-----------------------------------
git init
git remote add origin https://github.com/ViknanS/docker.git
git branch -M main

-----------------------------------
STEP 7 – ADD AND COMMIT
-----------------------------------
git add .
git commit -m "Docker CI setup"

-----------------------------------
STEP 8 – PUSH CODE
-----------------------------------
git pull origin main --rebase
git push -u origin main

-----------------------------------
STEP 9 – ADD GITHUB SECRETS
-----------------------------------
Go to GitHub repo → Settings → Secrets → Actions

Add:
DOCKER_USERNAME = your Docker Hub username
DOCKER_PASSWORD = Docker access token

-----------------------------------
STEP 10 – EXECUTION
-----------------------------------
1. Push code to GitHub
2. Go to repository → Actions tab
3. Workflow "Docker CI" will run automatically
4. It will:
   - Login to Docker Hub
   - Build Docker image
   - Push image to Docker Hub

-----------------------------------
OUTPUT
-----------------------------------
- Docker image created
- Image pushed to Docker Hub
- CI pipeline executed successfully

-----------------------------------
IMPORTANT NOTES
-----------------------------------
- Folder must be: .github/workflows/
- File name: ci.yml
- Dockerfile must be in root
- Secrets names must match exactly
